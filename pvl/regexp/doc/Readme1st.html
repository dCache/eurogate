Readme for Regular Expression

The need to use RegularExpressions

Not every drive schould handle every kind of request. There are two inputs that can match on each other or not.
The first one is given by the administator. He gives in a set of data (expression) for each Drive.  The second input (assignment) is made by the user. He produces the request.
If the values of the request object match the expression of the drive the request may be served by that drive. In other case it couldn't be handled. In this case this Request may fit to another drive.
 

How we use it

Both sets of data are strings. And both strings are checkt on corect syntax and compiled by parsers. The compiler for drive expressions delivers an unresolved tree. The output of the compiler for requestassigns pruduces a hashtable.  They are compared by a third tool. It results a Boolean. A TRUE means that the expression of the request fit the expression of the drive.

An example shows better how to use the compilers and the comperator.

The two Parsers  were generated by Sriram Sankars compilercompiler called Javacc. Javacc(Copyright (c) 1996-1999 Sun Microsystems, Inc. Copyright (c) 1997-1999 Metamata, Inc.).
Like lex and yacc Javacc is a tool to create compilers. But Javacc is written in and produces code in Java and  accepts LL(k) grammars as opposed to LALR grammars used by yacc.
